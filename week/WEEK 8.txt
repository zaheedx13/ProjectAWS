WEEK 8 – STEP-WISE LAB EXECUTION (Jenkins Pipeline Automation for Maven Projects)

--------------------------------------------------------
PHASE 0 — PREREQUISITES
--------------------------------------------------------

Step 1: Start Jenkins service (make sure Docker Desktop or local Jenkins service is ON)
Step 2: Open browser → Enter:
        http://localhost:8080
Step 3: Login using:
        Username: admin
        Password: (your admin password)

Expected: Jenkins Dashboard opens.

--------------------------------------------------------
PHASE 1 — AUTOMATION FOR MAVEN JAVA PROJECT
--------------------------------------------------------

Step 4: Click "New Item" (left side menu)
Step 5: Enter item name: MavenJava_Build
Step 6: Select "Freestyle Project"
Step 7: Click "OK"

---- CONFIGURE THE JOB ----
Step 8: In Description → type: Java Build demo
Step 9: Scroll to "Source Code Management"
Step 10: Select "Git"
Step 11: Paste Git Repository URL of Maven Java project:
         https://github.com/<username>/<maven_java_repo>.git
Step 12: Branch to build:
         */main  (or */master based on repository)

---- BUILD STEPS ----
Step 13: Scroll to "Build"
Step 14: Click "Add Build Step" → Select "Invoke top-level Maven targets"
Step 15: Select Maven version: MAVEN_HOME
Step 16: In Goals field → type:
         clean

Step 17: Click "Add Build Step" again → "Invoke top-level Maven targets"
Step 18: Goals:
         install

---- POST-BUILD ACTIONS ----
Step 19: Click "Add Post-build Action" → "Archive the artifacts"
Step 20: Files to archive: **/*

Step 21: Click "Add Post-build Action" → "Build other projects"
Step 22: In Projects to build → type: MavenJava_Test
Step 23: Select option: Trigger only if build is stable

Step 24: Click Apply → Save

--------------------------------------------------------
PHASE 2 — CREATE MAVEN JAVA TEST JOB
--------------------------------------------------------

Step 25: Click "New Item"
Step 26: Enter project name: MavenJava_Test
Step 27: Choose "Freestyle Project" → Click OK

---- CONFIGURE ----
Step 28: Description: Test demo
Step 29: Scroll to Build Environment
Step 30: Check: Delete workspace before build starts

---- BUILD STEPS ----
Step 31: Click "Add Build Step" → "Copy artifacts from another project"
Step 32: Project name: MavenJava_Build
Step 33: Build selector: Stable build only
Step 34: Artifacts to copy: **/*

Step 35: Click "Add Build Step" → "Invoke top-level Maven targets"
Step 36: Maven version: MAVEN_HOME
Step 37: Goals: test

---- POST-BUILD ----
Step 38: Add Post-Build Action → "Archive the artifacts"
Step 39: Files to archive: **/*

Step 40: Click Apply → Save

--------------------------------------------------------
PHASE 3 — CREATE BUILD PIPELINE VIEW FOR MAVEN JAVA
--------------------------------------------------------

Step 41: On Dashboard, click **+**
Step 42: Enter name: MavenJava_Pipeline
Step 43: Select: "Build Pipeline View"
Step 44: Click OK

Step 45: Under Pipeline Flow:
         Layout: Based on upstream/downstream relationship
         Initial Job: MavenJava_Build
Step 46: Click Apply → Save

--------------------------------------------------------
PHASE 4 — RUN PIPELINE
--------------------------------------------------------

Step 47: Click Run / Trigger icon in pipeline view
Step 48: Click small console icon (black box) to view build logs
Step 49: Check output is in GREEN for both jobs:
         ✔ MavenJava_Build
         ✔ MavenJava_Test

Pipeline is successful if test job triggers automatically and artifacts archived.

PHASE 7 — CREATE DEPLOY JOB
--------------------------------------------------------

Step 70: Create new item → Name: MavenWeb_Deploy
Step 71: Freestyle Project → OK

Step 72: Build Env → Check "Delete workspace before build starts"
Step 73: Add Build Step → Copy artifacts from another project
         Project name: MavenWeb_Test
         Build: Stable build only
         Artifacts to copy: **/*

Step 74: Post-build Action → Deploy WAR/EAR to a container
         WAR/EAR File: **/*.war
         Context path: webpath

Step 75: Add container → Tomcat 9.x remote
         Credentials:
         Username: admin
         Password: 1234
         Tomcat URL: http://localhost:8085/

Step 76: Apply → Save
